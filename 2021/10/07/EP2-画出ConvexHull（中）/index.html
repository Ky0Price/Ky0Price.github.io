    一点说明

    虽然最终我们可能需要三张相图或者说三个convexhull，纯uspex数据的，纯官方数据库（文献）的以及两者混合的。这也就是说，为了最后画图的方面方便，uspex和数据库的数据最好用两个或多个变量分别保存，否则后期再自己切割就很麻烦了。

    MP和pymatgen

    解决完了USPEX那边的数据后，接下来的事情可能就稍微有一点挑战性了，那就是整合数据库那边的资源。

    数据库的选择

    这里还是首推Materials Project，它不仅数据丰富，功能强大，而且UI也不那么反人类，更重要的是，他可以用pymatgen提供的API来批量摘取、处理数据。

    关于MP数据库和pymatgen的介绍和用法请参考刘锦程的博客：http://blog.wangruixing.cn/2019/06/12/pymatgen-api/

    我的使用感受

    MP确实没的说，功能强大、丰富又方便，ICSD虽然老牌但是不用科学上网或者湖大的图书馆数字资源接口恐怕很难登进去，我最想吐槽的其实是pymatgen，它的功能介绍和算例写得着实扯淡。本来想着能够用它一条龙解决数据库这边的工作，结果却是坐在图书馆里对着他的官方文档戴了一整天的痛苦面具，最后不得不转头用ASE来画convexhull，接着又引发了一些系列的问题，足以让我再写一篇教程。

    pymatgen的（不）正确打开方式

    pymatgen的功能足足有几十上百条，不过在这个阶段我们主要会用到它的两个功能

    API的使用

    在对它画相图的功能彻底绝望了之后，我主要用到的也就是他的API抓取功能了。虽然说在这个阶段我们能用它抓取的数据也就个位数条，为了把他转成ASE能看懂的格式，最后还是手动输入的数据。不过这不妨碍我们练习一下它的数据爬取功能。

    官方文档（推荐啃英文，网页翻译的结果更不像人话）：https://pymatgen.org/pymatgen.ext.matproj.html

    这里是我当时用到的的代码

    from pymatgen.ext.matproj import MPRester
with MPRester("YOUR_API_KEY") as m:  # 给人分享代码的时候一定注意给自己的API秘钥打码
    mp_entries = m.get_entries(chemsys_formula_id_criteria='Li-C-N',property_data='energy')
    # 在数据库里爬取所有的Li-C-N三元系统的结构，并打印
print(mp_entries)

    几点说明：

    • 可以看到我这里加了一条property_data='energy'这是因为它给的数据太全了，这是好事，但是数据格式比较“不那么具备适用性”，这就不是很好了，因为我们最终不需要用它来画相图，而是用ASE，所以我希望能够用这几条来规范一下它的输出，但事实证明这样没什么大用，效果还是这幅样子

    • 我们最终需要哪些条目

      • 结构对应的ID（可选）

      • 结构的分子式

      • 结构对应的能量（整个分子的，而且是Corrected的）

      • 结构的cif文件

      • 空间群

    如何分别获取这些条目

    1）可以看到前两个我们用API得到的数据里面就有，写程序摘出来或者手动复制粘贴即可

    2）关于能量，我不得不吐槽，我一开始看到energy（uncorrected）和energy（final）（见上图）的时候，以为final就是指的经过修正的能量，但当我看到每一条结构的这两条能量都一样的时候，我意识到了问题的严重性。当我试着直接去网页里面手动找的时候，我确认了自己心里的那点怀疑，就以上图的mp-9610为例：

    所以我最后还是手动一条条去网页上复制的。

    3）cif文件的获取可以用API，也可以直接去网页上下载，下载的方法详见刘锦程的B站教学《理论催化计算实战教学3-VASP输入文件》，获得cif文件之后可以用vesta直接一键转成POSCAR文件，方法详见《理论催化计算实战教学4-VASP输入文件》，cif文件用完也别丢，后头可能还要用他来找空间群，为什么用可能呢，是因为我在写这篇文章的时候依然没搞懂到底要怎么找

    4）空间群的获取也有两种方法，取网站上面手动复制粘贴，或者你可以研究一下pymatgen的SpacegroupAnalyzer()这个类，应该也能得到不错的结果，研究出来了可以联系我，把这个功能也加入脚本套餐，方便之后的数据整理

    为啥不用pymatgen来画相图

    准确来说，你可用它来画MP的那张convexhull，但是用它来接收USPEX的数据的时候，就要抓瞎了。

    我们来看看它给的如何结合自己已有的VASP计算数据和MP的数据来生成相图的。

    # “#”后面的是官方的注释，“##”后面的是我的吐槽
from pymatgen.ext.matproj import MPRester
from pymatgen.apps.borg.hive import VaspToComputedEntryDrone
from pymatgen.apps.borg.queen import BorgQueen
from pymatgen.entries.compatibility import MaterialsProjectCompatibility
from pymatgen.analysis.phase_diagram import PhaseDiagram, PDPlotter

# Assimilate VASP calculations into ComputedEntry object. Let's assume that
# the calculations are for a series of new LixFeyOz phases that we want to
# know the phase stability.
drone = VaspToComputedEntryDrone()
queen = BorgQueen(drone, rootpath=".")  ##所以这个rootpath我到底应该填啥，VASP计算完之后这么多结果文件，你到底要啥
## 最后我发现它要的是vasprun.xml文件，而USPEX虽然调用vasp做的计算，然而输出文件里面并没有这个，我还回头去翻了他那个一百多页的使用说明书，也没看到能通过加什么启动设置来得到这个文件，这也就是为什么我最后还是用了ASE去画相图
entries = queen.get_data() 
## 这些个drone，queen，Brog方法/变量名到底有什么深意，我并没有看懂，还特别不好记，官网查资料的时候也容易看晕

# Obtain all existing Li-Fe-O phases using the Materials Project REST API
with MPRester("USER_API_KEY") as m:
    mp_entries = m.get_entries_in_chemsys(["Li", "Fe", "O"])
## 它确实能获得所有的三元体系结构，但是如果你真的用MP网站上的画相图功能的时候，你会发现stable point里面不仅有三元结构，还有二元的，也就意味着我们最后还是要补充上这几个数据，或者说pymatgen还有什么功能我没发掘到，但至少这个指令不太行，我自己写的那个也一样
# Combined entry from calculated run with Materials Project entries
entries.extend(mp_entries)

# Process entries using the MaterialsProjectCompatibility
compat = MaterialsProjectCompatibility()
entries = compat.process_entries(entries)

# Generate and plot Li-Fe-O phase diagram
pd = PhaseDiagram(entries)
plotter = PDPlotter(pd)
plotter.show()

    相信我已经解释得蛮清楚了

    总结一下我们需要自己动手做的工作

    1）手动或者自动化地从MP网站里面摘取需要的结构信息，记得它有两部分

    贴一下MP网站的画相图功能

    2）由于MP里的结构数据默认是在0K0GPa下的，所以如果你做的是0GPa下的搜索，那么就不需要再做什么多余的事情了，但如果你做的是压力下的搜索，那么这些结构拿来之后还不能直接用，你要用他的cif转POSCAR文件拿去VASP里面算它在压力下的空间结构和能量信息等。至于怎么算，其实我也没试过，但想来应该就是在INCAR里面改几个参数，详情见VASP官网的指南或者VASP WiKi

    WiKi地址：https://www.vasp.at/wiki/index.php/The_VASP_Manual

    VASP 在线手册：https://cms.mpi.univie.ac.at/vasp/vasp/vasp.html（可能要科学上网）

    一个很有用的工具

    刚刚提到了用POSCAR文件做VASP计算，那就不得不介绍一下VASPKIT，是一个我个人认为功德无量的脚本软件，他其他的几十个功能我还没用过，但光是他自动生成VASP计算的输入文件就已经很香了（生成的依据就是你给他的POSCAR文件），而且通过bash语言还能实现批量生成输入文件（夹）。这种酸爽，只有自己亲自写过那四大输入文件的人才能体会。

    不过要注意的是他自动生成INCAR文件的时候要小心一点，最好参数自己稍微检查一下，比如我看他给C，N这种非金属原子生成的INCAR文件里面得SIGMA值也是0.05，这个值用来算金属是好的，但是对非金属和气体分子而言就不一定适合了，这里贴一下LVTHW里面的原话：

    SIGMA的取值和ISMEAR息息相关，也就是和ISMEAR这个参数同时出现，设定。下面是一些基本的介绍：

    • 如果用了ISMEAR = -5； SIGMA的值可以忽略，也可以不管。（对于所有体系都适用)；
    • 对于金属： ISMEAR = 1 或者0，非金属: ISMEAR= 0 的时候，一般取 SIGMA = 0.10 即可，默认值是0.20。不放心的话，用0.05。
    • 对于气体分子，原子体系（也就是你把分子或者原子放到一个box里面）：死死记住下面组合就可以了， ISMEAR = 0; SIGMA = 0.01。

    不过很多人都不放心，为了保险想测试一番。标准是： SIGMA的取值要保证OUTCAR 中的 entropy T*S 这一项，平均到每个原子上，要小于 1-2 meV。检查的时候可以用这个命令:

    grep 'entropy T'  OUTCAR

    关于最后这个命令，我试过把他改成其他形式，比如grep 'entropy T*S' OUTCAR之类的，效果远不如他写的这条，放心用就是了。
